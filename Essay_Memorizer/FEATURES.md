## 📐 布局优化详情

### 容器尺寸
```css
屏幕尺寸         内容宽度        说明
1920×1080   →   1600px       标准横屏（推荐）
1600×900    →   1400px       中等横屏
1440×900    →   1200px       小型横屏
<1024px     →   95%          平板和手机
```

### 内边距优化
- **页面外边距**：20px（减少浪费）
- **卡片内边距**：24px（阶段内容）
- **段落间距**：16-20px（紧凑但清晰）
- **按钮内边距**：10px 24px（易于点击）

### 滚动区域高度
所有可滚动区域都基于视口高度(vh)动态计算：
- **阶段一**：`max-height: calc(100vh - 400px)`
- **阶段二**：`max-height: calc(100vh - 420px)`
- **阶段三**：`max-height: calc(100vh - 480px)`

确保在1920×1080下，每个阶段的核心内容尽量不需要滚动。

## 🔷 阶段功能详解

### 阶段一：逻辑框架

#### 核心功能
1. **完整范文展示**
   - 按段落组织
   - 段落类型标签（Introduction, Body, Conclusion）
   - 灰色背景区分
   - 左侧紫色边框

2. **一键显示核心句**
   - 点击按钮切换显示/隐藏
   - 渐进式动画（逐个淡入）
   - 100ms延迟，营造逐层展示效果
   - 紫色渐变背景突出显示

3. **按钮状态**
   - 显示状态：眼睛图标 + "显示核心句"
   - 隐藏状态：斜眼图标 + "隐藏核心句"

#### 数据结构
- 5个段落（1引言 + 3主体 + 1结论）
- 每段包含：类型、完整文本、核心句

---

### 阶段二：句式拆解

#### 三种视图模式

**1. 原文视图**
- 纯净的原文展示

**3. 词汇扩展视图**

- 斜体显示同义词

#### 视图切换
- 平滑过渡动画
- 按钮状态联动
- 图例自动显示/隐藏
- 保持滚动位置

#### 数据规模
- **33个完整句子**
- **50+ 重点词汇**
- **150+ 同义词扩展**
- 每句包含：原文、SVO标注、HTML版本、词汇表

### 阶段三：背诵练习

#### 🆕 全新交互流程

**状态机设计**：
```
[初始状态] 
    ↓ 点击"开始背诵"
```

- 已显示的句子不会消失

  - 白色背景

**3. 进度跟踪系统**
- **进度条**：0-100%渐变填充
- **句子计数**：X/33 实时更新
- **百分比显示**：精确到个位数

**4. 按钮智能提示**
- 状态1：💡 "点击显示提示"（准备看SVO）
- 状态2：✅ "点击显示完整句子"（准备看原文）
- 图标和文字联动变化

**5. 完成庆祝**
- 🎉 庆祝图标动画
- 祝贺消息
- "再次练习"按钮
- 进度条显示100%

#### 辅助功能
- **重置按钮**：随时重新开始
- **空格键快捷键**：快速切换（可选）
- **平滑滚动**：自动定位到当前句子

## 🎯 交互设计亮点

### 1. 无需输入的设计理念
传统背诵系统需要学生输入文本，这会带来：
- ❌ 打字速度影响学习节奏
- ❌ 拼写错误导致挫败感
- ❌ 不适合投影仪教学

**我们的解决方案**：
- ✅ 纯点击式交互
- ✅ 教师可远程控制（投影仪）
- ✅ 学生专注于背诵本身
- ✅ 降低使用门槛

### 2. 渐进式提示
- 先给SVO框架（少量信息）
- 再显示完整句子（完整信息）
- 符合人类记忆曲线
- 提高背诵效率

### 3. 累积显示的教学价值
- 学生可以看到已背诵内容
- 增强成就感
- 便于复习前面的句子
- 老师可以看到整体进度

### 4. 实时反馈
- 进度条可视化
- 句子计数清晰
- 按钮文字动态变化
- 完成后立即庆祝

```javascript
{
    active: false,              // 是否处于背诵状态
    currentSentenceIndex: 0,    // 当前句子索引
    showingSVO: false,          // 是否正在显示SVO
```
